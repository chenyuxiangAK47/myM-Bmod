## 测试步骤

### 1. 基础功能测试
- [ ] **单位显示测试**
  - 在自定义战斗中选择每个测试单位
  - 确认单位图标、名称正确显示
  - 确认单位信息面板显示正确

- [ ] **单位属性测试**
  - 检查单位等级（应该是6级）
  - 检查单位分组（应该是Infantry）
  - 检查单位文化归属是否正确

### 2. 战斗测试
- [ ] **进入战斗**
  - 使用测试单位进入自定义战斗
  - 确认单位能正常生成
  - 确认单位模型正确显示

- [ ] **装备测试**
  - 检查单位是否装备了正确的武器
  - 检查单位是否装备了正确的防具
  - 确认装备在战斗中正常显示和使用

- [ ] **技能测试**
  - 检查单位技能值是否正确
  - 在战斗中测试单位表现是否符合技能值

### 3. 游戏模式测试
- [ ] **自定义战斗**
  - 测试所有7个派系的测试单位
  - 确认都能正常选择和使用

- [ ] **战役模式**（如果适用）
  - 检查测试单位是否出现在招募列表中
  - 测试是否能正常招募
  - 测试是否能正常升级

### 4. 日志检查
- [ ] **检查游戏日志**
  - 查看 `rgl_log_*.txt` 文件
  - 确认所有测试单位文件都已加载
  - 查找是否有错误或警告信息

2. **添加升级路径**

   - 每次只恢复一个派系，测试是否正常

## 常见问题排查

### 如果单位不显示：
1. 检查 `SubModule.xml` 中的路径是否正确
2. 检查 XML 文件格式是否正确
3. 检查游戏日志中的错误信息

### 如果单位显示但无法使用：
1. 检查装备ID是否存在
2. 检查技能值是否合理
3. 检查文化归属是否正确

### 如果单位在战斗中崩溃：
1. 检查装备ID是否正确
2. 检查模型引用是否正确
3. 检查是否有缺失的必需属性
